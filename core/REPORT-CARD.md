### Historical contract findings (found & fixed)

API-contract issues surfaced while integrating context.dev, and how we resolved them:

- **Endpoints need a /web/ path prefix** (`contract/path-prefix`) — Every call 403'd; the documented paths omit the /web/ segment. _Fix:_ Prefix paths with /web/ (e.g. /web/scrape/markdown).
- **Scrape is GET, not POST** (`contract/scrape-method`) — POST to the scrape endpoint failed; it is GET /web/scrape/markdown?url=. _Fix:_ Call scrape via GET with a url query param.
- **Extract result is wrapped under .data** (`contract/data-unwrap`) — Extract returns { status, url, data, metadata }; the object lives under .data. _Fix:_ Unwrap response.data after extract.
- **null inside a JSON-schema enum 500s the validator** (`contract/null-enum-500`) — A schema with null in an enum crashed the extractor (HTTP 500). _Fix:_ Use null-safe schemas ({type:["string","null"]}); never null in an enum.
- **Search bills per result and ignores limit** (`contract/search-billing`) — Web search rejects a limit param and always bills 10 credits (1 per result). _Fix:_ Budget-gate search at the full 10-credit cost.

## Methodology

Grading is offline and deterministic against a committed corpus of hand-checked ground-truth profiles. The live `--collect` step calls context.dev to refresh the recorded extractions, timing each call and reading credits from the ledger; grading then runs with no key. Name uses normalized exact match; one-liner/positioning use ground-truth token recall (hit ≥ 0.5); tags use exact-set F1; features use fuzzy (token-overlap) F1; links match on host. Numbers are reported as measured — no cherry-picking.
