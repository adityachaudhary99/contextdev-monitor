# Context.dev Monitor Vocabulary

Category: The market phrase a user asks to map, such as "web scraping APIs".
Player: A company, project, or product that plausibly competes in the category.
Profile: The typed, cited summary for one player: positioning, pricing, tags, and source links.
Landscape: The structured market map produced from a category and its player profiles.
Snapshot: A captured landscape at one point in time, reduced to diff-relevant fields.
Motion/Diff: The over-time comparison between snapshots: entrants, exits, pricing moves, and capability changes.
Report-card: The reproducible evaluation of extraction quality, latency, cost, and failure modes.
Agent-skill: The portable Claude-Code workflow that maps a category through context.dev MCP or the CLI fallback.
Demo key: The maintainer-funded mode for bounded public demos and curated pages.
BYO key: A user-supplied context.dev API key sent server-side for one request and not stored.
Curated landscape: A checked-in landscape page that can be statically rendered and indexed.
On-demand landscape: A generated landscape created from the UI or API for a user-entered category.
Evidence: Source URLs and extracted facts attached to claims so readers can audit the result.
